# YOLOv3 / YOLOv4 单页复现实验设计

## 目标

在现有无人机 AI 知识库中新增可持续扩展的“实验 · 复现”板块，并首批提供 YOLOv3、YOLOv4 两个教学级完整复现实验。

每种网络只占一个高信息密度页面。页面同时提供完整可运行代码、详细中文注释、关键数学解释、运行命令、预期输出和常见错误，不再把一个网络拆成多节课程。

## 已确认的产品形态

- 新增顶级导航“实验 · 复现”，与“概念 · 理论”“专题 · 项目”等现有分组并列。
- 新增一个简短的实验板块总览页，用于列出所有网络复现；以后可继续加入 ResNet、Faster R-CNN、Transformer 等实验。
- YOLOv3 和 YOLOv4 各自使用一个独立页面，不合并成一个页面。
- 页面采用适合反复查阅的紧凑技术文档布局，不建设 YOLO 专属的多层课程导航。
- Notebook 体验页属于后续工作，不纳入本次实现。

## 交付结构

```text
experiments/
└── yolo_classics/
    ├── README.md
    ├── requirements.txt
    ├── yolov3_reproduction.py
    ├── yolov4_reproduction.py
    └── tests/
        ├── test_yolov3.py
        ├── test_yolov4.py
        └── test_smoke_pipeline.py

wiki/
└── experiments/
    ├── index.md
    ├── experiment-yolov3-reproduction.md
    └── experiment-yolov4-reproduction.md
```

`yolov3_reproduction.py` 和 `yolov4_reproduction.py` 分别是可直接运行的自包含教学实现。除 PyTorch、Pillow、NumPy 等公开依赖外，单个实现不依赖仓库中的隐藏业务模块。这样读者只打开一个网络页面或一个源码文件，就能看到完整的数据、模型、损失、训练、评估和推理闭环。

测试代码单独存放，避免挤占实验正文，但实验页面会说明每类测试验证了什么。

## 单页内容结构

每个网络页面按照以下顺序组织：

1. 网络定位与结构速览：输入输出、主干、特征融合、检测头和该版本的核心特点。
2. 数学与张量契约：边框参数化、锚框匹配、损失函数、解码过程以及关键张量形状。
3. 完整复现代码：展示对应 Python 文件的全部内容，不用省略号代替实现。
4. 代码解读：解释重要类和函数的职责、数据流、设计原因以及初学者容易混淆的细节。
5. 运行方法：环境安装、VOC/YOLO 数据目录、训练、验证、单图推理和断点恢复命令。
6. 输出解读与排错：损失项、mAP、检测结果、NaN、空预测、显存不足和标签错误。

源码中的每个类和函数必须有中文文档注释。涉及坐标系、广播、张量变形、标签分配、损失计算和 NMS 的复杂代码块，需要增加行内中文注释，说明“输入是什么、变换后是什么、为什么这样做”。普通赋值不添加无意义的逐行翻译。

## 源码与网页同步

网页中的完整代码以源码文件为唯一事实来源，构建阶段通过受限的代码包含标记读取并渲染指定文件，避免网页代码与实际可运行文件发生漂移。

代码包含功能只允许读取仓库内明确允许的实验目录，拒绝绝对路径和 `..` 路径穿越。构建测试需要覆盖正常包含、文件不存在和非法路径三种情况。

## YOLOv3 实现范围

- Darknet-53 主干及残差块。
- 三尺度 FPN 检测结构；416×416 输入时对应 13×13、26×26、52×52 输出。
- `tx, ty, tw, th`、目标置信度和类别概率的预测、标签分配与解码。
- 锚框匹配、边框回归、目标置信度 BCE 和类别 BCE。
- VOC XML 与通用 YOLO 文本标注读取。
- 随机缩放、平移、水平翻转和 HSV 扰动。
- 训练、验证、VOC AP/mAP、NMS、单图推理、检查点保存与恢复。

## YOLOv4 实现范围

- CSPDarknet-53、CSP 连接和 Mish 激活。
- SPP、PAN 与三尺度检测头。
- CIoU 边框损失、目标置信度 BCE、类别 BCE 和标签平滑。
- Mosaic、随机仿射、HSV 扰动和水平翻转。
- 与 YOLOv3 相同的数据接口、训练入口、评价口径和推理输出格式。
- 相同输入尺寸与数据划分下的公平对比说明。

## 训练与命令行行为

两个源码文件使用一致的命令行习惯，至少支持：

- `train`：训练、验证、保存最近与最佳检查点。
- `eval`：载入检查点并计算 VOC AP/mAP。
- `detect`：对单张图片执行预处理、推理、NMS 和结果绘制。
- `inspect`：打印网络输出形状，并在微型输入上执行一次前向和损失计算。

默认训练使用 SGD、Momentum、权重衰减、学习率预热和余弦退火，支持固定随机种子、CPU/GPU 自动选择、梯度裁剪和断点续训。参数非法或数据不完整时应快速失败，并给出包含文件位置或参数名称的中文错误信息。

## 错误处理

需要显式检测并解释以下情况：

- 数据目录、图片、标注或类别文件不存在。
- VOC XML 或 YOLO 文本标注格式错误、类别越界、边框宽高非正数。
- 类别数、检测头输出通道和检查点参数不兼容。
- 锚框数量、顺序或尺度配置无效。
- 训练损失出现 NaN 或 Inf。
- 推理没有保留任何检测框。
- CUDA 不可用或显存不足。

## 验证标准

自动化验证不要求在本机完成完整 VOC 长周期训练，但必须证明实现闭环成立：

- IoU、CIoU、坐标转换、标签分配、预测解码和 NMS 单元测试通过。
- YOLOv3 与 YOLOv4 三尺度输出形状正确。
- 两个模型均能完成损失前向与反向传播，梯度为有限值。
- 微型合成数据能够完成训练、保存检查点、重新加载和单图推理。
- VOC XML 与 YOLO 文本格式均有最小读取测试。
- 站点构建通过，“实验 · 复现”导航、两个实验页面、源码包含和页面内链接均有效。
- 网页页面在桌面和移动视口下无内容重叠，长代码块可横向滚动，不挤破正文布局。

## 明确不做

- 本次不实现 COCO JSON 数据读取与 COCO 官方评测。
- 本次不实现多 GPU 分布式训练、ONNX 导出或 TensorRT 部署。
- 本次不下载或提交大型数据集、训练权重和完整训练产物。
- 本次不创建 Notebook 体验页。
- 本次不把基础目标检测知识拆成多个独立教学页面。

## 后续扩展原则

后续每增加一种网络，只增加一个自包含源码文件、一个高密度实验页面和对应测试，并登记到实验总览。Notebook 体验页应调用已有源码，而不是复制另一套模型实现。
