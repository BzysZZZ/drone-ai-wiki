# Rich Notes Implementation Plan

> **For agentic workers:** Execute inline only. Do not use subagents. Do not run git commands.

**Goal:** Add a private Word-like rich text notes page and make the reference-library card view denser.

**Architecture:** Reuse the existing Flask session login and SQLite database. Store rich notes in a new `rich_notes` table as HTML documents. Add a generated `notes.html` page with dedicated vanilla JS/CSS assets.

**Tech Stack:** Python 3, Flask, SQLite, unittest, static HTML/CSS/vanilla JavaScript, `document.execCommand` for lightweight rich text editing.

**Git Policy:** Do not run `git add`, `git commit`, or `git push`.

---

## File Structure

- Modify `server/app.py`: add `rich_notes` table, note helpers, authenticated note CRUD API, content/title limits.
- Modify `server/test_app.py`: add note API tests for login requirement, create/list/get/update/delete, size validation.
- Create `wiki/notes.md`: generated page entry.
- Modify `build_site.py`: add notes title/group and inject notes CSS/JS only on `notes.md`.
- Create `assets/notes.js`: Word-like private rich-text editor UI.
- Create `assets/notes.css`: note list/editor/toolbar layout.
- Modify `assets/references.css`: make reference cards more compact.
- Generated by build: `site/notes.html`, `site/assets/notes.js`, `site/assets/notes.css`.
